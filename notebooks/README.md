# notebooks/

> Notebooks de investigación exploratoria. Cualquier notebook que
> produzca código útil debe terminar en `src/trading_bot/...` siguiendo
> SDD; los notebooks NO sustituyen a la documentación ni a los tests.

## Convención

- Nombre: `<YYYY-MM-DD>-<autor>-<tema>.ipynb`.
- Cabecera Markdown con: objetivo, hipótesis, resultado, decisiones.
- Si se descubre un patrón reutilizable, promociónalo a un módulo
  mediante el flujo SDD (`01-requirements` → `02-bdd` → ...).
